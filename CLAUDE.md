# CLAUDE.md

This file provides guidance to Claude Code when working with this repository.

## Project Goal

A clean, minimal art gallery website for Shala, hosted on GitHub Pages at **[shalaball.com](https://shalaball.com)**.

## Tech Stack

- **Site:** Plain HTML + CSS + minimal JS. No build tools or frameworks.
- **Admin:** Node.js + Express (local only, not deployed). Start with `cd art-gallery/admin && npm start`.
- **Admin dependencies:** express, multer, sharp, heic-convert

## Content Management

**CONTENT.md is the source of truth** for all page names and photo labels.

- labels.js format: `{ filename, title, desc, zoom? }` on all pages
- labels.js is auto-generated by the admin on every Save
- img src uses `../photos/filename` (relative path) on all gallery pages

## Gallery Pages

- All pages use unified CSS with both one-column and two-column layout rules
- Layout is controlled by `class="gallery"` (one-column) or `class="gallery two-column"` on the gallery div
- Layout can be toggled per-page from the admin via the Layout dropdown (One Column / Two Column)
- Mobile: two-column pages collapse to single column at ≤768px

## Admin UI (<http://localhost:3000>)

- **Save Changes** — writes CONTENT.md, rebuilds labels.js, updates nav
- **Sync Website** — git pull --rebase + push (deploys to GitHub Pages)
- **Preview** — auto-saves, then opens preview tab
- **Layout dropdown** — toggle One Column / Two Column per page (updates live site HTML)
- **Download button** — on each photo card and library card, downloads the original file
- **Rotate buttons** (↺ ↻) — rotate photo in-place via sharp
- **Zoom buttons** (− +) — scale thumbnail, saved to labels.js
- Drag-and-drop reordering via SortableJS
- HEIC upload: auto-converted to JPEG via heic-convert

## App Launchers

- **Art-gallery.command** — double-click in Finder, opens Terminal window
- **Art-gallery-restart.command** — kills running server and restarts it
- **ArtAdmin.app** — Automator app, no Terminal window; on first run prompts for admin folder path, saves to `~/.artgalleryadmin`

## Deployment

GitHub Pages auto-deploys on every push to the `master` branch.
Custom domain `shalaball.com` configured via CNAME file and GitHub Pages settings.
